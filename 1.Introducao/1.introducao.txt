Cenário Geral : QUESTÕES - PREPARAÇÃO - EXPLORAÇÃO - CONCLUSÃO - COMUNICAÇÃO

1.Questões :
O passo inicial, com a escolha do conjunto de dados a ser usado. 
Deve haver uma boa definição do problema a ser resolvido e, assim, traçar a melhor estratégia para solucioná-lo, definindo as técnicas estatísticas e de machine learning que serão empregadas, sempre guiando seu processo pela questão: 
qual o valor e impacto do que eu estou fazendo?

Sites que disponibilizam datasets gratuitos : Data.world, Kaggle, FiveThiertyEight, BuzzFeed, Socrata Open Data, Data.gov, Quandl, Reddit, UCI Repository, Academic Torrents, portal Brasileiro de Dados Abertos, IBGE.

2.Preparação :
Na preparação é feito limpeza dos dados, removendo-se os principais erros e inconsistências no conjunto de dados e garantindo que os dados estejam prontos para análise.
Aqui será usado a ferramenta de ETL mais usada no mercado, o Integration Services da Microsoft, organização de dados com Pandas, em Python e será visto como o Excel pode estragar os dados.

3.Exploração :
Nessa etapa, uma boa análise descritiva permite formular hipóteses e questões relevantes, utilizando medidas estatísticas (média, variância, correlação) e visualização (histogramas, box plots, gráficos de dispersão).Nessa fase, o cientista de dados :
3.1.sabe a natureza de cada variável (qualitativa ou quantitativa);
3.2.sabe qual a distribuição de cada uma;
3.3.descobre se existem e quem são os outliers;
3.4.identifica a relação entre as variáveis.
Muita coisa acontece nessa fase, mas o mais importante é buscar identificar nos dados, objetos que outras pessoas não perceberam e ajustar, posteriormente, os modelos.
Essa é a fase de acostumar-se com os dados de forma a realizar duas perguntas de maneira correta.

4.Conclusão :
As conclusões são, geralmente tiradas com a aplicação de Machine Learning. Existem scripts prontos para praticamente todo tipo de modelo.

5.Comunicação
Toda comunicação é realizada através de gráficos e Storytelling. Será usado gráficos em R e Phyton.
Você pode construir ótimos modelos, mas deve ser capaz de explicá-los de forma clara e objetiva, ou boa parte do seu trabalho pode ser perdida. Com os passos anteriores feitos, você contará uma história.

“Great data scientists are great storytellers”.


OBS.:
Visualização Interativa dos Dados
Use gráficos apropriados para cada tipo de variável. Tente apresentar gráficos que capturem os olhos, mas que sejam simples e compreensíveis. 
Visualizações interativas incluem ferramentas como dashboards, que são uma maneira rápida e fácil de interagir com os dados.
Uma boa visualização permite comunicar de forma clara o que há nos dados e o leva a ter insights adicionais.
